# Evaluation Guide

Use this page to evaluate whether OpenLIT Ops Cloud fits a real workflow.

## What To Test

- LLM observability dashboard
- OpenLIT Ops Cloud
- OpenLIT Ops Cloud documentation
- OpenLIT Ops Cloud SaaS
- openlitops workflow

## Expected Evidence

- Open OpenLIT Ops Cloud and choose a bounded pilot workflow.
- Enter public-safe examples for LLM observability dashboard.
- Review the generated queue, evidence, or approval state with the owner.
- Use pricing and checkout links when the pilot needs a paid workspace.
- Export the resulting report or evidence for the team record.

## Risk Checks

- Do not put API keys, tokens, payment details, private logs, or customer records in public issues.
- Use public-safe sample data for examples and directory submissions.
- Treat generated receipts and scores as reviewer evidence, not as a substitute for accountable human approval.

## Buyer Path

Default plan: team.
